# Git Commit 规范

## 提交格式

```
<type>(<scope>): <description>

<optional body>
```

**示例：**
```
feat(人工): 创建README.md并整理我的理解

- 添加了API通知系统的背景描述
- 整理了对需求的理解和边界分析
```

## 类型 (type)

| 类型 | 说明 | 示例                     |
|------|------|------------------------|
| `feat` | 新功能 | `feat(人工): 添加用户注册回调通知` |
| `fix` | Bug 修复 | `fix(AI): 修复重复扣费问题`    |
| `refactor` | 重构 | `refactor(AI): 重构投递逻辑` |
| `docs` | 文档更新 | `docs(AI): 更新API文档`        |
| `test` | 测试相关 | `test(AI): 添加单元测试`         |
| `chore` | 构建/工具 | `chore(AI): 升级依赖版本`        |
| `perf` | 性能优化 | `perf(AI): 优化查询性能`         |
| `ci` | CI/CD | `ci(AI): 配置GitHub Actions` |

## 范围 (scope)

使用括号包裹，标识影响的功能模块：

- `人工` - 完全人工的输出
- `AI` - AI辅助的输出

**注意**：完全不可以省略这个范围，否则提交会被拒绝。
**注意**：scope中可以包含多个，例如 `feat(人工, AI): ...`，但不建议过多混合。

## 描述 (description)

- 使用**中文**描述
- 简洁明了，不超过 50 字符
- 描述"做什么"而非"怎么做"
- 结尾不加句号

**Good:**
```
feat(人工): 添加重试机制
fix(AI): 修复登录超时问题
```

**Bad:**
```
feat: 添加了重试机制的功能（中文句号，描述过长）
fix: 修复了bug（太模糊）
```

## 正文 (body)

当需要详细说明时，在描述下方空一行后编写：

- 说明**为什么**做这个改动
- 说明**如何**实现（如果非显而易见）
- 列出具体的变更点

每条 bullet point 以 `-` 开头。

## 提交示例

```
feat(人工): 添加API通知系统

- 新增通知路由模块，接收业务系统请求
- 新增消息队列，实现异步投递
- 支持多种认证方式配置

fix(AI): 修复超时重试逻辑

- 指数退避策略实现错误
- 最大重试次数未生效
```

## 自动触发

当提交代码时，确保：
1. 类型正确（参考类型表）
2. 描述简洁、中文
3. 涉及多模块时使用 `refactor` 而非混合多个类型

## 文档语言规则

**所有文档输出必须使用中文**，包括但不限于：
- README 文档
- 设计文档
- 提交信息（description 和 body）
- 注释
- 代码中的字符串（用户可见的部分）

**例外**：技术术语、变量名、代码等保持英文。

## 在 Claude Code 中使用

每次提交代码前，先确认提交信息符合此规范：

```
<type>(<scope>): <description>
```

例如提交一个修复：
```
fix(通知): 修复重试队列丢失问题
```
