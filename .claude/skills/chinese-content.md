# 中文内容规则

## 核心规则

**所有新建文件的内容必须使用中文**，包括：
- Markdown 文档（README、设计文档、规格文档等）
- 代码注释
- 提交信息
- 配置文件中的注释
- 日志消息

**例外（保持英文）：**
- 技术术语（如 API、HTTP、SQL 等通用术语）
- 变量名、函数名、类名
- 代码字符串（用户不可见的内部逻辑）
- 数学公式、算法描述

## 判断标准

如果用户能看到的内容，必须用中文：
- 文件顶部的描述性文字 → 中文
- 函数上方的文档注释 → 中文
- README 中的说明 → 中文
- 配置项的解释 → 中文

## 示例

**正确：**
```markdown
# 用户认证模块

这个模块负责处理用户的登录和注册功能。
```

**正确（代码文件）：**
```go
// FindUserByID 根据用户ID查询用户信息
func FindUserByID(id string) (*User, error) {
    // 查询逻辑
}
```

**错误：**
```markdown
# User Authentication Module
```

## 在 Claude Code 中

创建任何新文件前，确保内容使用中文编写，除非是：
- 技术术语
- 代码标识符
- 用户不可见的内部字符串
